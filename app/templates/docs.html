1. 认证与鉴权

TwinSentry 支持两种认证方式，分别用于 Web 管理和自动化 Agent：

1.1 自动化 Agent 认证

TwinSentry 区分任务阶段，请根据 Agent 类型使用对应的 Key：

分析智能体 (Analysis Agent):

X-Agent-Key: YOUR_ANALYSIS_KEY

处置智能体 (Disposition Agent):

X-Agent-Key: YOUR_DISPOSITION_KEY

1.2 Web 用户 Bearer 认证

用于管理端接口（如获取告警列表）。获取登录 Token 后在 Header 中使用：

Authorization: Bearer <your_token>

2. 告警接入 (Webhook)

通用告警接收端。您可以从 SIEM (Splunk, ELK) 或 EDR 发送 POST 请求将告警推送到 TwinSentry。

POST http://your-server/api/webhook/receiver
请求体 (JSON):
{
  "title": "[严重] 恶意进程执行告警",
  "text": "主机名: WS-01\n进程名: mimikatz.exe\n用户: SYSTEM\n路径: C:\\Temp\\"
}

3. Agent 交互接口

3.1 拉取待分析任务 (Analysis)

GET /analysis/fetch

从队列中原子的获取一条 PENDING 状态的告警。

3.2 提交分析结果 (Analysis)

POST /analysis/submit 
{
  "id": 123,
  "result": "经研判，该告警为真实攻击，建议封禁恶意源 IP。"
}

3.3 拉取待处置任务 (Process)

GET /disposition/fetch

从队列中拉取一条已完成分析且待处置的告警。

3.4 提交处置结果 (Process)

POST /disposition/submit 
{
  "id": 123,
  "result": "已根据分析建议封禁源IP: 1.1.1.1"
}

4. 系统管理接口

用于第三方看板或管理系统集成。

路径 方法 描述
/api/alerts GET 获取所有活动告警列表
/api/stats/dashboard GET 获取仪表盘统计数据 (饼图/趋势)
/api/alerts/<id> DELETE 逻辑删除指定告警
