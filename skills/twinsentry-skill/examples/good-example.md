# 优秀使用示例 (Good Example)

本示例展示了在 TwinSentry 当中，优秀的智能体（Analysis / Action Agent）应该如何格式化输出它的分析结论与处置建议。

## 场景：检测到 Web 注入攻击 (SQLi)

### 1. 优秀的分析结论 (Analysis Log)
在 `submit_result` 时提交的 `analysis_log`，将被 TwinSentry 前端捕获并呈现：

```markdown
**威胁研判：高危 (High)**

经过综合分析，确认源 IP `107.150.119.229` 正在对我们的 `cloud.hesc.info` 发起持续的 SQL 注入攻击。

**分析详情：**
1. **攻击特征**：HTTP 请求参数中包含了典型的 SQL 盲注 Payload（如 `%20AND%201=1` 和 `WAITFOR DELAY`）。
2. **频次分析**：在过去 5 分钟内，该源 IP 产生了超过 150 次 404/500 报错，属于高频扫描与探测。
3. **资产重要性**：受攻击域名属于核心业务节点。

**处置建议（供 Action Agent 参考）：**
- 立即在 WAF/防火墙 层面封禁 IP `107.150.119.229`。
- 提取并清查该 IP 在过去 24 小时内是否有成功返回 200 的历史请求。
```
*(优点：Markdown 格式清晰，使用了加粗、列表，并且明确给出了行动建议与风险定性。)*

### 2. 优秀的处置结论 (Action Log)
在处置完成后，通过 `submit_result` 提交的 `action_log`：

```markdown
**处置动作已完成**

根据分析智能体的研判结果，已执行以下自动化防御编排动作：

- [x] **WAF 阻断**：已成功调用内部 WAF 接口，将 IP `107.150.119.229` 加入全局防御黑名单，封禁时长：**永久**。
- [x] **工单流转**：已自动创建追踪工单 `SEC-2026-9012` 并指派至安全运营组。
- [x] **日志核查**：在监控系统中下发了过去 24 小时的 200 状态码行为核查任务，暂未发现失陷行迹。

*处置智能体生命周期执行完毕。*
```
*(优点：使用了 Markdown 任务待办列表，清晰反馈了具体的阻断节点和执行的工单记录，非常方便运维人员进行事后审计。)*
