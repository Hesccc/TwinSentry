# 安全规则与最佳实践

## 访问控制规则

### Token 管理

| 规则 | 说明 |
|------|------|
| Token 最小权限 | 仅授予 API Token IP 黑名单写入权限，避免使用管理员全权 Token |
| Token 定期轮换 | 建议每 90 天更换一次 API Token |
| 不硬编码 Token | Token 必须存于 `.env` 文件或系统环境变量，禁止写入代码文件 |
| `.gitignore` | `.env` 文件必须加入 `.gitignore`，禁止提交到版本控制 |

### 网络访问

| 规则 | 说明 |
|------|------|
| 限制 API 访问来源 | 雷池管理后台仅应向内网或可信 IP 开放，不要暴露到公网 |
| SSL 警告 | 本工具使用 `verify=False` 跳过自签名证书验证，适用于内网，若有条件请使用受信任证书 |

## IP 封禁规则

### 封禁前确认

1. **确认 IP 归属** — 避免误封内网 IP 或 CDN 出口节点
2. **确认 group_id** — 封禁前在雷池后台确认目标 IP 组是否正确
3. **记录备注** — 使用 `--comment` 参数记录封禁原因，便于事后审计

### 覆盖式 API 注意事项

雷池 IP 组 API 是**完全覆盖式**接口，每次调用会替换组内全部 IP：

```
⚠️  PUT /api/open/ipgroup 会清空原有 IP 并替换为新列表
```

**安全操作流程**：
1. 先查询现有组内 IP（在雷池后台或通过 GET API）
2. 将新 IP 合并到现有列表
3. 提交完整列表

## IP 格式规范

### 支持的格式

| 格式 | 示例 | 说明 |
|------|------|------|
| IPv4 | `192.168.1.1` | 单个 IPv4 地址 |
| IPv6 | `2001:db8::1` | 单个 IPv6 地址 |
| IPv4 CIDR | `10.0.0.0/8` | IPv4 网段 |
| IPv6 CIDR | `2001:db8::/32` | IPv6 网段 |

### 不支持的格式

| 格式 | 示例 | 建议 |
|------|------|------|
| 主机名 | `malicious.example.com` | 先解析为 IP 再封禁 |
| IP 范围 | `192.168.1.1-100` | 转换为 CIDR 格式 |
| 带端口 | `1.2.3.4:80` | 仅支持 IP 地址，不含端口 |

> IP 格式由雷池 API 负责验证；提交不支持的格式将返回 HTTP 400。

## 审计与监控建议

1. **定期审查黑名单** — 建议每月审查 IP 组内容，移除过期条目
2. **设置告警** — 对 `error_code = CONNECTION_ERROR` 设置监控告警，防止封禁失败
3. **版本控制配置** — 将 `.env.example` 纳入版本控制（不含真实 Token）
